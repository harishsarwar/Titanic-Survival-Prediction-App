# 🚢 Titanic Survival Prediction App

> A full-stack Machine Learning web application that predicts Titanic passenger survival — built with FastAPI, SQLAlchemy, Docker, and Scikit-learn. Covers the complete ML lifecycle from model training to production deployment.

---

## 🔗 Links

- 🌐 **Live App:** https://titanic-survival-prediction-app-l1gm.onrender.com
- 📘 **API Docs:** https://titanic-survival-prediction-app-l1gm.onrender.com/docs

## ✅ Features

- 🔮 **ML Prediction** — predicts Titanic survival using a trained ML model pipeline
- 👤 **User System** — register as new user or login with existing user ID
- 📋 **View Predictions** — see all past predictions in a clean table
- ✏️ **Edit Records** — update any existing prediction record
- 🗑️ **Delete Account** — remove all records from the database
- 📡 **REST API** — full API with interactive Swagger docs at `/docs`
- 🐳 **Dockerized** — runs in Docker, no manual environment setup needed
- 💾 **Dual Database** — SQLite for development, PostgreSQL for production

## 🤖 ML Pipeline

```
Raw Data (Titanic CSV)
        ↓
Data Ingestion
→ load dataset
→ train / test split
→ save to artifacts/
        ↓
Data Transformation
→ handle missing values (imputation)
→ encode categorical features
→ scale numerical features
→ save preprocessor.pkl
        ↓
Model Training
→ train multiple models
   (Random Forest, XGBoost, Decision Tree...)
→ hyperparameter tuning (GridSearchCV)
→ evaluate on test set
→ select best model
→ save model.pkl
        ↓
Prediction Pipeline
→ load model.pkl + preprocessor.pkl
→ transform input data
→ return prediction
        ↓
FastAPI Endpoint
→ receive form data
→ run prediction pipeline
→ save result to database
→ display to user
```

## 📡 API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/` | Home page |
| GET | `/register` | Register / Login form |
| POST | `/register` | Submit registration |
| GET | `/predict` | Prediction form |
| POST | `/predict` | Submit prediction → run ML model → save to DB |
| GET | `/predictions/{user_id}` | View all predictions for a user |
| GET | `/edit/{id}` | Edit prediction form |
| POST | `/edit/{id}` | Submit updated prediction |
| POST | `/delete/{user_id}` | Delete all records for a user |

> 📘 Full interactive API docs: **`/docs`** (Swagger UI)

## 🗄️ Database Schema

```sql
CREATE TABLE predictions (
    id         INTEGER  PRIMARY KEY AUTOINCREMENT,
    user_id    VARCHAR  NOT NULL,
    name       VARCHAR  NOT NULL,
    pclass     INTEGER,
    sex        VARCHAR,
    age        FLOAT,
    sibsp      INTEGER,
    parch      INTEGER,
    fare       FLOAT,
    embarked   VARCHAR,
    prediction VARCHAR
);
```

## 🚀 Run Locally

### Option 1 — Docker (recommended)

```bash
# clone the repo
git clone https://github.com/YOUR_USERNAME/titanic-ml-app.git
cd titanic-ml-app

# build and start
docker-compose up --build

# visit
http://localhost:8000
```

### Option 2 — Without Docker

```bash
# clone the repo
git clone https://github.com/YOUR_USERNAME/titanic-ml-app.git
cd titanic-ml-app

# create virtual environment
python -m venv venv
source venv/bin/activate        # Mac/Linux
venv\Scripts\activate           # Windows

# install dependencies
pip install -r requirements.txt

# train the model (first time only)
python -m src.components.data_ingestion

# run the app
python app.py

# visit
http://localhost:8000
```
